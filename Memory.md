## 1、What to Store

### 1.1 Memory Types（记忆类型）

从“人类认知科学（Cognitive Science）”映射到 AI Agent 的记忆架构。

**Query**:为什么不把所有的记忆每一条作为一个记录全部塞入一个DB里面然后做retriever，而是去做分类存储？ 
**Answer**:检索污染，但是更细致的待TODO肯定不好，为什么不好？

- Semantic Memory（语义记忆）  
这是用来记录**事实知识库**，里面会存世界知识（常识）、用户偏好、长期设定、概念关系等等  
**举例：**  
1、该用户名字：阿雄  
2、用户工作：程序员  
3、用户喜好：简介输出、抽象概念等等  
**该类的特点：**  
长期稳定、去时间化、可抽象、可泛化
- Episodic Memory（情景记忆）  
这个是用来记录：**经历（回忆）**，里面大概会记录：用户在什么时间段干什么活工作？在什么时间段关注哪一块内容？执行什么程序遇到什么bug？  
**举例：**  
1、2026-05-24:用户讨论了 CRF 与 Transformer  
2、2026-04-30:用户关注 “具身智能”领域相关知识  
**该类的特点：**  
时间性强、上下文依赖、不稳定（越来越多）
- Procedural Memory（程序记忆）
 用来记录之前搜索过的“技能”。
 **举例：**
 1、如何写 SQL
 2、如何调用 API
 3、如何部署 Docker
 4、如何推理数学题
 **该类的特点：** 
 强动作性、可执行、可复用、流程化
- Working Memory（工作记忆）
 短期记忆:当前任务的上下文
 **举例：**  
 1、当前用户的问题  
 2、中间推理Tools 或者 CoT、ToT的结果  
 **该类的特点：** 
 短期、容量有限、任务绑定
- Shared Memory（共享记忆）  
 这是 Multi-Agent 时代出现的产物。是多个Agent共享信息的"白板"。  
 **举例：**  
 1、团队目标、共享任务状态  
 2、共享知识、协同计划  
 **该类的特点：**  
 多Agent之间的访问、一致性问题、权限问题、协同问题
 **Tips**
上面的额5种类型其实在边界上划分是存在一些模糊地带的。并且是可以转换的，比如：
Episodic（情景） → Semantic（语义）：用户多次要求C++ -> 抽象为 用户偏好
Semantic（语义） → Procedural（程序）：用户希望SQL出答案->关于查询的直接给SQL
Working（短期记忆） → Episodic（情景）：当前对话结束 -> 历史记录
